# 🎤 Interview Preparation Guide

Mastering the interview is a skill unto itself. It requires a combination of technical depth, clear communication, and problem-solving intuition.

---

## 🛠️ The 4 Pillars of Tech Interviews

### 1. Coding (DSA)
- **Strategy**: Don't just jump into code. Clarify the problem, discuss the approach, trace small examples, and then implement.
- **Focus**: Common patterns like Sliding Window, Two Pointers, DFS/BFS, and Backtracking.
- **Tools**: [LeetCode](https://leetcode.com/), [HackerRank](https://www.hackerrank.com/), [NeetCode](https://neetcode.io/).

### 2. System Design
- **Strategy**: Use the "Requirement → Estimation → API → Schema → High Level → Detailed Sync" flow ($RES$ flow).
- **Focus**: Load balancers, Databases, Caching, Sharding, and Consistency vs Availability (CAP Theorem).
- **Tools**: System Design Primer, Designing Data-Intensive Applications (DDIA).

### 3. Object-Oriented Design (OOD)
- **Strategy**: Follow the SOLID principles. Define Classes, Relationships (Association, Composition, Inheritance), and Design Patterns (Singleton, Factory, Observer).
- **Focus**: Parking Lot design, Elevator design, Amazon/Library system.

### 4. Behavioral & Culture Fit
- **Strategy**: Use the **STAR Method** (Situation, Task, Action, Result).
- **Focus**: Tell me about a time you had a conflict, a time you failed, or your biggest technical challenge.
- **Company Values**: Research the company's leadership principles (e.g., Amazon's LPs).

## 📅 Preparation Timeline

| Timeline | Goal | Task |
| :--- | :--- | :--- |
| **Month 1** | Foundation | Review DSA basics and basic patterns. |
| **Month 2** | Practice | Solve 150-200 medium LeetCode problems. |
| **Month 3** | Archetecture | Study System Design and OOD patterns. |
| **Month 4** | Polish | Mock interviews, Behavioral prep, and Resume polishing. |

---

## 💡 Pro-Tips
- **Think Out Loud**: The interviewer wants to see *how* you think, not just the solution.
- **Ask Clarifying Questions**: Constraints (size of input, memory limits) often change the optimal solution.
- **Clean Code Matters**: Even in an interview, use meaningful variable names and modularize your code.
