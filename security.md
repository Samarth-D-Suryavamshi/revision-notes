# Security Policy

## Supported Versions

The latest version of this repository is actively maintained.

| Version         | Supported |
| --------------- | :-------: |
| Latest (`main`) |     ✅     |
| Older versions  |     ❌     |

---

# Reporting a Security Vulnerability

If you discover a security vulnerability related to this repository, please **do not open a public GitHub Issue**.

Instead, report the issue privately through GitHub or contact the repository maintainer directly.

Please include the following information:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Suggested mitigation (if available)
* Screenshots or logs (if applicable)

Reports containing sufficient technical details help ensure faster investigation and resolution.

---

# Scope

This security policy applies to:

* GitHub Pages deployment
* GitHub Actions workflows
* MkDocs configuration
* Repository configuration
* Static documentation website
* Repository infrastructure

This policy does **not** apply to third-party services or dependencies maintained by external projects.

---

# Security Best Practices

This repository follows several security practices, including:

* Automated deployment through GitHub Actions
* Version-controlled documentation
* Public review of repository changes
* Dependency management through `requirements.txt`
* GitHub repository security features when available

---

# Responsible Disclosure

Please provide maintainers with a reasonable period to investigate and resolve reported security issues before making public disclosures.

Responsible disclosure helps protect users while fixes are being prepared.

# Third-Party Dependencies

* MkDocs

Security issues affecting these projects should also be reported to their respective maintainers when appropriate.
